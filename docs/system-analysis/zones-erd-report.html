10. جدول Cardinality — العلاقات بالأرقام

مرجع سريع للعلاقات بين الكيانات

من إلى النوع FK / Pivot
المستخدمالصالة (مدير)1:1stations.manager_id
المستخدمالصالة (موظف)N:1users.station_id
الصالةالباقة1:Npackages.station_id
الصالةالجهاز1:Ndevices.station_id
الباقةالجهاز1:Ndevices.package_id
الصالةالحجز1:Nbookings.station_id
الحجزالدفعة1:1payments.booking_id UQ
الصالةالخدمةN:Mservice_station
البطولةالمشارك1:Ntournament_participants.tournament_id
البطولةالمباراة1:Ntournament_matches.tournament_id
عطل الجهازمصروف الصالة1:1hall_expenses.device_fault_id UQ
تعليق الصالةردود1:Nstation_comments.parent_id
طلب الانضمامالصالة1:1hall_join_requests.station_id
المستخدمالدورN:Mmodel_has_roles

One To One (1:1)

  • الحجز ↔ الدفعة
  • عطل الجهاز ↔ مصروف الصالة
  • المستخدم ↔ الصالة المُدارة
  • طلب الانضمام ↔ الصالة المُنشأة

Many To Many (N:M)

  • الصالة ↔ الخدمة
  • المستخدم ↔ الدور
  • المستخدم ↔ الصلاحية
  • الدور ↔ الصلاحية
